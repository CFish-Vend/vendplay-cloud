=== STRIPE WEBHOOK BEHAVIOR (CRITICAL) ===

- Stripe may deliver the SAME webhook event multiple times
- Delivery can be:
    • immediate duplicate
    • delayed (seconds or minutes later)

- Each Stripe event has a unique event_id

FIX IMPLEMENTED:
- Backend stores processed event_id in processed_events table
- Before queue_vend():
    → check if event_id already exists
    → if yes → IGNORE webhook
    → if no → process and store event_id

RULE:
- queue_vend() must ONLY run once per event_id
- NEVER rely on vend_queue state to prevent duplicates

RESULT:
- Prevents:
    • double vend (immediate)
    • phantom vend (delayed)
